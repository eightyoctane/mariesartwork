Blue Mountain Recipe Database and Restaurant Guide Once you get into it, I think you will find this application quite sophisticated and capable. Of course the price for this functionality is some level of complication.
How to use it

We're giving you the highest authority here (which means that if you misuse it, you can screw up the database for everybody). The highest privilege login is admin/passwd which allows you to change anything. I've done this because the database is largely unnpopulated, and I'm wanting you to contribute as much as possible. Please sit down with me (Gary), and get checked out (we can use terms like this at BMA, because people actually understand them!) so you can be effective with the program without messing it up.

First step is to login with admin/passwd, and create yourself a lower privilege account (I suggest "edit" level) with your name. It's important to do this because there is "Meal Planner" and "Shopping List" functionality included in the application. You want meals that you plan associated with your login account. It all looks pretty cool but you tell me. You'll still need to log in as admin periodically to add and change fields - you'll see what I mean as you get into it.

Recipes
Researching this and thinking about it a bit, a recipe can range anywhere from "let's throw it together from memory", to abstract chemistry. So how can we contribute to this database without making it overwhelmingly complicated? Start simple. If you have a fantastic recipe or idea you want to share with other Blue Mountain residents, get it on here as simply and quickly as possible. You don't have to fill in all the fields - that can happen later - or never, meaning that we all talk frequently, and someone can give you a call if they need more detail. The main thing here I think, is we all need more recipe ideas, so let's help each other out.
